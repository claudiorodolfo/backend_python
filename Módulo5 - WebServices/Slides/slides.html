Recapitulação - HTTP Protocol

Protocolo fundamental para comunicação web.

  • Métodos: GET, POST, PUT, DELETE, PATCH
  • Status codes: 2xx (sucesso), 3xx (redirect), 4xx (cliente), 5xx (servidor)
  • Request: método, URL, headers, body
  • Response: status, headers, body
  • Stateless: cada requisição é independente
Slide 46 de 54

Recapitulação - REST Architecture

Representational State Transfer - arquitetura para APIs.

  • Recursos identificados por URIs
  • Métodos HTTP mapeiam operações
  • Stateless: sem estado no servidor
  • Cacheable quando apropriado
  • Uniform interface
Slide 47 de 54

Recapitulação - JSON

JavaScript Object Notation - formato de dados.

  • Serialização: objeto Python → JSON string
  • Desserialização: JSON string → objeto Python
  • Bibliotecas: json (nativo), marshmallow, pydantic
  • Validação de schema
  • Format padrão para APIs REST
Slide 48 de 54

Recapitulação - Frameworks Web

Frameworks para criar APIs em Python.

  • Flask: minimalista, flexível
  • FastAPI: moderno, rápido, async, documentação automática
  • Django REST Framework: completo, baseado em Django
  • Escolher baseado em necessidades do projeto
  • Todos suportam REST e JSON
Slide 49 de 54

Recapitulação - Endpoints e Rotas

Estrutura de URLs e mapeamento de rotas.

  • URLs descritivas e hierárquicas
  • Parâmetros de rota e query string
  • Validação de entrada
  • Versionamento de API (/v1/, /v2/)
  • Nomes de recursos em plural (GET /users/)
Slide 50 de 54

Recapitulação - Autenticação

Mecanismos para proteger APIs.

  • JWT: tokens stateless, seguro
  • OAuth 2.0: autorização entre serviços
  • Basic Auth: simples, menos seguro
  • Headers: Authorization Bearer token
  • Permissões e roles (RBAC)
Slide 51 de 54

Recapitulação - Tratamento de Erros

Como lidar com erros em APIs.

  • Códigos HTTP apropriados (400, 401, 404, 500)
  • Mensagens de erro claras e úteis
  • Estrutura consistente de respostas de erro
  • Logging adequado (sem expor informações sensíveis)
  • Tratamento de exceções centralizado

Recapitulação - Documentação

Importância de documentar APIs.

  • Swagger/OpenAPI: especificação padrão
  • Documentação automática (FastAPI, DRF)
  • Postman: coleções para testes
  • Exemplos de request/response
  • Manter documentação atualizada

  • Frameworks: Flask, FastAPI, DRF
