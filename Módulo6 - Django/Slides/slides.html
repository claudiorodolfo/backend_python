Recapitulação - Django Fundamentals

Framework web completo "batteries included".

  • Filosofia: DRY (Don't Repeat Yourself)
  • MVT: Model-View-Template (similar ao MVC)
  • Estrutura de projeto organizada
  • Admin interface automática
  • Rápido desenvolvimento, código limpo
Slide 57 de 72

Recapitulação - Models e ORM

Object-Relational Mapping do Django.

  • Models definem estrutura de dados
  • Campos: CharField, IntegerField, DateTimeField, ForeignKey, etc.
  • Relacionamentos: ForeignKey, ManyToMany, OneToOne
  • Migrations: versionamento do schema
  • Queries: ORM gera SQL automaticamente
Slide 58 de 72

Recapitulação - Views

Views processam requisições e retornam respostas.

  • Function-based views: simples e diretas
  • Class-based views: reutilizáveis e poderosas
  • Generic views: CRUD pré-construído
  • Context: dados passados para templates
  • Decorators: @login_required, @csrf_exempt
Slide 59 de 72

Recapitulação - Templates

Django Template Language (DTL) para renderização.

  • Sintaxe: {{ variável }}, {% tag %}
  • Filtros: {{ value|upper }}, {{ value|date }}
  • Herança de templates: {% extends %}
  • Includes: {% include %}
  • Templates reutilizáveis e organizados
Slide 60 de 72

Recapitulação - URLs e Roteamento

Mapeamento de URLs para views.

  • URLconf: urlpatterns
  • URL patterns com regex ou path converters
  • Namespaces: app_name para organização
  • Reverse URLs: url 'app:view' com {% url %}
  • URLs descritivas e RESTful
Slide 61 de 72

Recapitulação - Forms

Sistema robusto de formulários do Django.

  • Django Forms: validação automática
  • Model Forms: baseados em models
  • CSRF Protection: segurança integrada
  • Widgets customizados
  • Clean methods para validação customizada
Slide 62 de 72

Recapitulação - Autenticação

Sistema de autenticação integrado.

  • User Model padrão ou customizado
  • Login/Logout views e decorators
  • Permissões: can, has_perm
  • Groups: agrupamento de permissões
  • @login_required, @permission_required
Slide 63 de 72

Recapitulação - Django REST Framework

Framework para criar APIs REST com Django.

  • Serializers: conversão model ↔ JSON
  • ViewSets e Routers: CRUD automático
  • Permissions: controle de acesso
  • Authentication: JWT, Token, Session
  • Pagination: controle de resultados
Slide 64 de 72

Recapitulação - Middleware

Processamento de requests/responses.

  • Executado em cada requisição
  • Common: sessões, autenticação, CSRF
  • Custom middleware possível
  • Ordem importa
  • Útil para logging, cache, etc.
Slide 65 de 72

Recapitulação - Signals

Sistema de notificações entre componentes.

  • pre_save, post_save, pre_delete, etc.
  • Conectores: @receiver decorator
  • Útil para ações automáticas
  • Evitar lógica de negócio em signals
  • Documentar bem quando usar
Slide 66 de 72

Recapitulação - Testes

Framework de testes integrado.

  • TestCase: base para testes
  • Fixtures: dados de teste
  • Client testing: simular requisições
  • Testes de models, views, forms
  • python manage.py test
Slide 67 de 72

Recapitulação - Static e Media Files

Gerenciamento de arquivos estáticos e uploads.

  • STATIC_URL, STATIC_ROOT, STATICFILES_DIRS
  • MEDIA_URL, MEDIA_ROOT para uploads
  • collectstatic para produção
  • FileField e ImageField em models
  • Servir arquivos em desenvolvimento vs produção
Slide 68 de 72

Recapitulação - Settings

Configuração do projeto Django.

  • settings.py: configuração principal
  • Ambientes: dev, staging, prod
  • Variáveis de ambiente para secrets
  • DEBUG = False em produção
  • ALLOWED_HOSTS configurado
Slide 69 de 72

Recapitulação - Deploy

Colocando aplicação Django em produção.

  • Settings de produção: DEBUG=False, ALLOWED_HOSTS
  • Static files: collectstatic, whitenoise ou CDN
  • Otimização: cache, database indexes
  • Segurança: HTTPS, CSRF, secrets
  • Deployment: Gunicorn, Nginx, PostgreSQL

Recapitulação - Boas Práticas Django

Princípios para projetos Django bem estruturados.

  • Apps organizados por funcionalidade
  • Reutilizar apps do Django e comunidade
  • Migrations versionadas no Git
  • Testes para funcionalidades críticas
  • Documentação e código limpo
