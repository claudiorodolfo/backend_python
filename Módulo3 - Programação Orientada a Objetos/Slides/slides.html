Recapitulação - 4 Pilares da POO

Os quatro conceitos fundamentais da Programação Orientada a Objetos.

  • Encapsulamento: ocultar detalhes internos, expor interface
  • Abstração: representar conceitos do mundo real
  • Herança: reutilizar código e especializar classes
  • Polimorfismo: diferentes formas de executar a mesma ação
Slide 31 de 54

Recapitulação - Classes e Objetos

Classes são modelos (plantillas) que definem atributos e comportamentos.

  • Classes definem estrutura e comportamento
  • Objetos são instâncias concretas de classes
  • __init__: método construtor
  • __str__ e __repr__: representação textual
  • Propriedades (@property) e métodos
Slide 32 de 54

Recapitulação - Métodos Especiais

Métodos especiais (dunder methods) definem comportamentos do objeto.

  • __init__(self): construtor
  • __str__(self): representação legível (str())
  • __repr__(self): representação técnica (repr())
  • @classmethod: métodos da classe
  • @staticmethod: métodos utilitários
Slide 33 de 54

Recapitulação - Encapsulamento

Ocultar detalhes de implementação e expor apenas o necessário.

  • Atributos privados com _ (convenção) ou __ (name mangling)
  • Getters e setters via @property
  • Métodos públicos e privados
  • Interface clara e estável
  • Proteção de dados internos
Slide 34 de 54

Recapitulação - Herança

Permite criar classes filhas que herdam características das classes pais.

  • Classe base (pai) e classe derivada (filha)
  • super() para acessar classe pai
  • Override de métodos
  • Reutilização de código
  • Hierarquia de classes
Slide 35 de 54

Recapitulação - Polimorfismo

Diferentes classes podem implementar o mesmo método de formas diferentes.

  • Métodos com mesmo nome, comportamentos diferentes
  • Interfaces comuns
  • Duck typing em Python
  • Flexibilidade e extensibilidade
  • Abstração de implementação
Slide 36 de 54

Recapitulação - Relacionamentos

Diferentes tipos de relacionamentos entre classes.

  • Associação: classes relacionadas, independência
  • Agregação: parte-todo, objeto pode existir sem o todo
  • Composição: parte-todo, objeto depende do todo
  • Herança: relação is-a (é um tipo de)
Slide 37 de 54

Recapitulação - Organização de Código

Estruturar código em módulos e pacotes para melhor organização.

  • Módulos: arquivos .py
  • Pacotes: diretórios com __init__.py
  • Separação de responsabilidades
  • Arquitetura em camadas
  • import e from ... import
Slide 38 de 54

Recapitulação - Arquitetura em Camadas

Separação clara de responsabilidades em diferentes camadas.

  • Model: representação de dados
  • DAO (Data Access Object): acesso a banco
  • Service: lógica de negócio
  • Controller: interface com usuário/API
  • Baixo acoplamento, alta coesão
Slide 39 de 54

Recapitulação - Padrão DAO

Data Access Object encapsula acesso a dados.

  • Separa lógica de acesso a dados
  • Abstração de operações de banco
  • Facilita manutenção e testes
  • Um DAO por entidade (UserDAO, ProductDAO)
  • Métodos: create, read, update, delete
Slide 40 de 54

Recapitulação - Service Layer

Camada de serviços contém lógica de negócio.

  • Orquestra operações complexas
  • Validações de regras de negócio
  • Coordena múltiplos DAOs
  • Transações e operações atômicas
  • Interface entre Controller e DAO
Slide 41 de 54

Recapitulação - Repository Pattern

Abstrai acesso a dados de forma mais genérica.

  • Abstração de coleções de objetos
  • Independente de implementação (SQL, NoSQL)
  • Facilita testes (mock repositories)
  • Interface comum para diferentes fontes
  • Similar ao DAO, mais genérico
Slide 42 de 54

Recapitulação - Singleton

Padrão que garante apenas uma instância da classe.

  • Instância única em todo o programa
  • Útil para conexões, configurações
  • Controle de acesso a recursos compartilhados
  • Implementação via __new__ ou decorador
  • Use com cuidado (pode dificultar testes)

Recapitulação - Projeto Prático

Integração de POO com banco de dados SQLite.

  • Model Layer: classes que representam entidades
  • DAO Layer: classes para acesso a dados
  • Service Layer: lógica de negócio
  • Separação clara de responsabilidades
  • Código organizado e manutenível
Slide 44 de 54

Recapitulação - Benefícios da POO

Por que usar Programação Orientada a Objetos.

  • Reutilização de código (herança, composição)
  • Organização e estruturação
  • Manutenibilidade e extensibilidade
  • Modelagem próxima ao mundo real
  • Facilita trabalho em equipe
Slide 45 de 54

Recapitulação - Boas Práticas

Princípios para escrever código POO de qualidade.

  • Classes com responsabilidade única
  • Nomes descritivos e claros
  • Documentação (docstrings)
  • Encapsulamento adequado
  • Evitar herança profunda (máx 2-3 níveis)
Slide 46 de 54

Recapitulação - Python e POO

Características específicas de POO em Python.

  • Duck typing: "se anda como pato, é pato"
  • Métodos mágicos (dunder methods)
  • Múltipla herança suportada
  • Mixins para funcionalidades compartilhadas
  • ABC (Abstract Base Classes) para interfaces
Slide 47 de 54

Recapitulação - Abstração

Simplificar complexidade focando no essencial.

  • Classes abstratas (ABC)
  • Interfaces implícitas (duck typing)
  • Ocultar detalhes de implementação
  • Focar no que objeto faz, não como
  • Facilita entendimento e uso
Slide 48 de 54

Recapitulação - Relacionamentos Práticos

Como aplicar relacionamentos em projetos reais.

  • Composição sobre herança (quando possível)
  • Associações flexíveis (fácil de modificar)
  • Agregação para coleções
  • Herança para relacionamentos is-a
  • Analisar necessidade antes de decidir
Slide 49 de 54

Recapitulação - Testes e POO

Como testar código orientado a objetos.

  • Mock objects para dependências
  • Testes unitários por classe
  • Testes de integração entre camadas
  • Isolamento de responsabilidades facilita testes
  • unittest e pytest frameworks
Slide 50 de 54

Recapitulação - Padrões Arquiteturais

Organização de código em projetos maiores.

  • MVC: Model-View-Controller
  • Arquitetura em camadas
  • Dependency Injection
  • Service-Oriented Architecture
  • Escolher padrão adequado ao projeto

Recapitulação - Erros Comuns

Pequenos erros a evitar em POO.

  • Classes muito grandes (god classes)
  • Herança desnecessária
  • Acoplamento forte entre classes
  • Ignorar encapsulamento
  • Misturar responsabilidades
