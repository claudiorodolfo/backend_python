Recapitulação - Bancos Relacionais

Bancos de dados relacionais armazenam dados em tabelas com relacionamentos bem definidos.

  • Estrutura de tabelas, linhas e colunas
  • Chaves primárias e estrangeiras
  • Integridade referencial
  • Operações CRUD (Create, Read, Update, Delete)
Slide 35 de 54

Recapitulação - SQLite

SQLite é um banco de dados leve e embutido, perfeito para desenvolvimento e aplicações pequenas.

  • Zero-configuração: não requer servidor
  • Baseado em arquivo único
  • Biblioteca sqlite3 nativa do Python
  • Context managers para gerenciamento automático de conexões
  • Comandos PRAGMA para configuração
Slide 36 de 54

Recapitulação - MySQL

MySQL é um SGBD relacional robusto e amplamente usado em produção.

  • Requer instalação e configuração de servidor
  • Bibliotecas: mysql.connector e pymysql
  • Credenciais via variáveis de ambiente
  • Prepared statements para segurança
  • Pools de conexão para performance
Slide 37 de 54

Recapitulação - PostgreSQL

PostgreSQL é um SGBD relacional open-source avançado e extensível.

  • Instalação e configuração personalizável
  • Driver psycopg2 para Python
  • Gerenciamento avançado de esquemas
  • Backup e restore com pg_dump/pg_restore
  • Monitoramento de performance
Slide 38 de 54

Recapitulação - Segurança

A segurança é fundamental ao trabalhar com bancos de dados.

  • Variáveis de ambiente para credenciais
  • Prepared statements previnem SQL injection
  • Gestão segura de credenciais
  • Permissões e roles apropriados
  • Nunca hardcode senhas no código
Slide 39 de 54

Recapitulação - Operações Básicas

Operações fundamentais para trabalhar com bancos de dados.

  • Conexão: estabelecer e fechar conexões corretamente
  • Queries: SELECT, INSERT, UPDATE, DELETE
  • Gestão de recursos: sempre fechar conexões
  • Context managers garantem fechamento automático
Slide 40 de 54

Recapitulação - Transações ACID

Transações garantem consistência dos dados.

  • Atomicidade: tudo ou nada
  • Consistência: dados sempre válidos
  • Isolamento: transações não interferem
  • Durabilidade: mudanças persistem
  • Commit e Rollback para controle
Slide 41 de 54

Recapitulação - Gestão de Recursos

Importante gerenciar recursos adequadamente para evitar vazamentos.

  • Sempre fechar conexões (try/finally ou context manager)
  • Cursor.close() e Connection.close()
  • Pools de conexão para aplicações multithread
  • Context managers (with statement) são recomendados
Slide 42 de 54

Recapitulação - Otimização

Boas práticas para melhorar performance de queries.

  • Índices em colunas frequentemente consultadas
  • Evitar SELECT * (selecionar apenas colunas necessárias)
  • Usar WHERE apropriadamente
  • Limitar resultados quando possível
  • Analisar queries lentas (EXPLAIN)
Slide 43 de 54

Recapitulação - Escolhendo o Banco

Critérios para escolher o banco de dados adequado.

  • SQLite: desenvolvimento, protótipos, apps simples
  • MySQL: web apps, média escala, fácil setup
  • PostgreSQL: apps complexos, dados críticos, extensibilidade
  • Considere: escala, features necessárias, suporte
Slide 44 de 54

Recapitulação - Bibliotecas Python

Principais bibliotecas para trabalhar com bancos de dados em Python.

  • sqlite3: nativo, para SQLite
  • mysql-connector-python: oficial MySQL
  • pymysql: alternativa para MySQL
  • psycopg2: driver PostgreSQL
  • SQLAlchemy: ORM unificado (próximos módulos)
Slide 45 de 54

Recapitulação - Padrões de Código

Padrões recomendados ao trabalhar com bancos de dados.

  • Separar lógica de acesso a dados (DAO pattern)
  • Usar context managers para conexões
  • Prepared statements para todas as queries
  • Tratamento de exceções adequado
  • Código limpo e bem documentado
Slide 46 de 54

Recapitulação - Tratamento de Erros

Como lidar adequadamente com erros em operações de banco.

  • try/except para capturar exceções específicas
  • OperationalError: problemas de conexão
  • IntegrityError: violações de integridade
  • ProgrammingError: erros de sintaxe SQL
  • Sempre fazer rollback em caso de erro
Slide 47 de 54

Recapitulação - Migrations

Gerenciamento de mudanças no esquema do banco.

  • Versionar mudanças no esquema
  • Scripts SQL para migrações
  • Backup antes de migrar
  • Testar em ambiente de desenvolvimento primeiro
  • Documentar todas as mudanças
Slide 48 de 54

Recapitulação - Boas Práticas

Princípios fundamentais para trabalhar com bancos de dados.

  • Nunca confiar em dados de entrada (validar sempre)
  • Usar transações para operações relacionadas
  • Fazer backup regularmente
  • Monitorar performance
  • Documentar estrutura do banco
Slide 49 de 54

Recapitulação - Performance

Estratégias para melhorar performance de aplicações com banco de dados.

  • Índices em colunas de busca frequente
  • Evitar N+1 queries (usar JOIN)
  • Cache de queries frequentes
  • Paginação para grandes resultados
  • Connection pooling em produção

Recapitulação - Integração com Python

Como integrar bancos de dados com código Python de forma eficiente.

  • Classes de acesso a dados (DAO)
  • Encapsular lógica de banco
  • Reutilizar conexões quando possível
  • Separar configuração do código
  • Testes com bancos de dados de teste
